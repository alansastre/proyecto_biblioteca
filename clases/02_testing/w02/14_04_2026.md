## Queries derivadas
* Spring Data JPA genera queries automáticamente a partir del **nombre del método**
* `findByActiveTrue` → `SELECT * FROM restaurantes WHERE active = TRUE`
* No tenemos que escribir SQL

Ejemplos:
```java
List<Restaurant> findByName(String name);
List<Restaurant> findByActiveTrue();
List<Restaurant> findByActiveFalse();
List<Restaurant> findByAveragePriceGreaterThan(Double price);
List<Restaurant> findByNameContainingIgnoreCase(String text);
long countByActiveTrue();
```

## Práctica: queries y tests
Añadir en `RestaurantRepository`:

```java
List<Restaurant> findByActiveTrue();
List<Restaurant> findByNameContainingIgnoreCase(String text);
List<Restaurant> findByAveragePriceGreaterThan(Double price);
long countByActiveTrue();
```

Y un test por cada uno en `RestaurantRepositoryTest`:

* [ ] `findByActiveTrue`: 2 activos + 1 inactivo → debe devolver 2
* [ ] `findByNameContainingIgnoreCase("paco")` con "Pacos Bar", "Casa Pepe", "Bar Paco" → debe devolver 2
* [ ] `findByAveragePriceGreaterThan(20.0)` con precios variados → solo los > 20
* [ ] `countByActiveTrue` → solo cuenta los `active = true`

Estructura recomendada:
```java
@Test
@DisplayName("findByActiveTrue devuelve solo restaurantes activos")
void testFindByActiveTrue() {
    // Given
    Restaurant r1 = Restaurant.builder().name("Activo 1").active(true).build();
    Restaurant r2 = Restaurant.builder().name("Activo 2").active(true).build();
    Restaurant r3 = Restaurant.builder().name("Inactivo").active(false).build();
    restaurantRepository.saveAll(List.of(r1, r2, r3));

    // When
    List<Restaurant> activos = restaurantRepository.findByActiveTrue();

    // Then
    assertEquals(2, activos.size());
    assertTrue(activos.stream().allMatch(Restaurant::getActive));
}
```
