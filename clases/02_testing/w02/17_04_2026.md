## 2. Entidad Order (pedido)

```java
package com.restaurantes.model;

import jakarta.persistence.*;
import lombok.*;
import java.time.LocalDateTime;

@Entity
@Getter
@Setter
@Builder
@NoArgsConstructor
@AllArgsConstructor
@ToString
@Table(name = "orders")
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private LocalDateTime date;
    private Double totalPrice;
    private Double tip;
    private Integer tableNumber;
    private Integer numPeople;

    @Enumerated(EnumType.STRING)
    private OrderStatus status;

    @ToString.Exclude
    @ManyToOne
    private Restaurant restaurant;
}
```

* `@Table(name = "orders")` — `order` es palabra reservada en SQL (`ORDER BY`), si no la renombramos da error
* `LocalDateTime` — incluye fecha y hora, a diferencia de `LocalDate` que solo tiene fecha
* `@ToString.Exclude` en restaurant para evitar recursión

## 3. Entidad OrderLine (línea de pedido)

Un pedido tiene varias líneas. Cada línea dice qué plato se pide y cuántas unidades:

```
Pedido mesa 5:
  - 2x Ensalada (5.00 €)    → línea 1
  - 1x Lentejas (8.00 €)    → línea 2
  - 3x Tarta de queso (7.50 €) → línea 3
```

```java
package com.restaurantes.model;

import jakarta.persistence.*;
import lombok.*;

@Entity
@Getter
@Setter
@Builder
@NoArgsConstructor
@AllArgsConstructor
@ToString
public class OrderLine {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Integer quantity;

    @ToString.Exclude
    @ManyToOne
    private Dish dish;

    @ToString.Exclude
    @ManyToOne
    private Order order;
}
```

* Primera entidad con **dos** `@ManyToOne`: hacia Dish (qué plato) y hacia Order (a qué pedido pertenece)

## 4. Repositorios

```java
package com.restaurantes.repository;

import com.restaurantes.model.Order;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;

public interface OrderRepository extends JpaRepository<Order, Long> {

    @Query("SELECT SUM(ol.quantity * ol.dish.price) FROM OrderLine ol WHERE ol.order.id = ?1")
    Double calculateTotalPrice(Long orderId);
}
```

```java
package com.restaurantes.repository;

import com.restaurantes.model.OrderLine;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.List;

public interface OrderLineRepository extends JpaRepository<OrderLine, Long> {

    List<OrderLine> findByOrderId(Long orderId);
}
```

### ¿Por qué `calculateTotalPrice` va en OrderRepository?

Aunque el `FROM` de la query es `OrderLine`, la respuesta es una propiedad del pedido (su total). La query se pone en el repositorio de la entidad sobre la que preguntamos:

| Query | Devuelve | Sujeto de la pregunta | Dónde va |
|---|---|---|---|
| `findByOrderId` | `List<OrderLine>` | "las líneas de este pedido" → OrderLine | `OrderLineRepository` |
| `calculateTotalPrice` | `Double` | "el total de este pedido" → Order | `OrderRepository` |

Cuando el service/controlador haga `orderRepository.calculateTotalPrice(id)`, lee natural: "tengo un id de pedido, quiero su total".

* `findByOrderId` — consulta derivada: filtra las líneas de un pedido concreto
* `calculateTotalPrice` — `@Query` con JPQL: la base de datos calcula directamente `SUM(cantidad × precio)`

## Servicio y test de servicio (Opcional)

Si nos da tiempo, introducimos una nueva capa: el **service**. Si no, lo dejamos para la semana que viene.

### ¿Por qué un service?

En una app real de restauración (tipo Glovo, carta digital en mesa, TPV del camarero), el flujo es:

```
Usuario añade plato → se guarda la línea → se recalcula el total del pedido → se muestra el nuevo total
```

Esto no es "una operación de base de datos" suelta: son **varias operaciones coordinadas** con una regla de negocio (el total = suma de líneas). Esto no debería vivir:

* En el controlador (el controlador solo recibe peticiones HTTP, no hace lógica)
* En el repositorio (el repositorio solo hace operaciones de BD sueltas)
* En la entidad (la entidad es un contenedor de datos)

**Ahí es donde entra el Service**: clase que coordina varias operaciones y aplica reglas de negocio.

```
Controller     ── capa HTTP (recibe peticiones, devuelve vistas)
   │
   ▼
Service        ── capa de lógica de negocio (cálculos, coordinación)
   │
   ▼
Repository     ── capa de base de datos (operaciones SQL)
   │
   ▼
Entity         ── los datos
```

### Crear OrderService

Nuevo archivo: `src/main/java/com/restaurantes/service/OrderService.java`

```java
package com.restaurantes.service;

import com.restaurantes.model.*;
import com.restaurantes.repository.*;
import org.springframework.stereotype.Service;

@Service
public class OrderService {

    private final OrderRepository orderRepository;
    private final OrderLineRepository orderLineRepository;
    private final DishRepository dishRepository;

    public OrderService(OrderRepository orderRepository,
                        OrderLineRepository orderLineRepository,
                        DishRepository dishRepository) {
        this.orderRepository = orderRepository;
        this.orderLineRepository = orderLineRepository;
        this.dishRepository = dishRepository;
    }

    /**
     * Recalcula el total del pedido sumando todas sus líneas y lo guarda.
     * Si el pedido no tiene líneas, el total queda en 0.
     */
    public Order recalculateTotal(Long orderId) {
        Order order = orderRepository.findById(orderId)
                .orElseThrow(() -> new RuntimeException("Pedido no encontrado: " + orderId));

        Double total = orderRepository.calculateTotalPrice(orderId);
        order.setTotalPrice(total != null ? total : 0.0);

        return orderRepository.save(order);
    }

    /**
     * Añade un plato al pedido con la cantidad indicada y recalcula el total.
     * Simula el flujo real: el camarero/cliente pulsa "+ Ensalada" y el total se actualiza.
     */
    public Order addDish(Long orderId, Long dishId, Integer quantity) {
        Order order = orderRepository.findById(orderId)
                .orElseThrow(() -> new RuntimeException("Pedido no encontrado: " + orderId));
        Dish dish = dishRepository.findById(dishId)
                .orElseThrow(() -> new RuntimeException("Plato no encontrado: " + dishId));

        OrderLine line = OrderLine.builder()
                .order(order)
                .dish(dish)
                .quantity(quantity)
                .build();
        orderLineRepository.save(line);

        return recalculateTotal(orderId);
    }
}
```

* `@Service` — le dice a Spring que esta clase es un componente de lógica de negocio
* Constructor — Spring inyecta los 3 repositorios automáticamente
* `recalculateTotal` — usa el `@Query` del repo para sumar y guarda el total en la entidad
* `addDish` — coordina tres operaciones: buscar pedido, buscar plato, crear línea, recalcular total

### Testear el servicio con @SpringBootTest

Hasta ahora hemos usado `@DataJpaTest` para testear repos. Para testear el service que usa **varios** repos, usamos `@SpringBootTest` con `@Transactional` (para rollback).

Nuevo archivo: `src/test/java/com/restaurantes/service/OrderServiceTest.java`

```java
package com.restaurantes.service;

import com.restaurantes.model.*;
import com.restaurantes.repository.*;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

import static org.junit.jupiter.api.Assertions.*;

@SpringBootTest
@Transactional
class OrderServiceTest {

    @Autowired OrderService orderService;
    @Autowired OrderRepository orderRepository;
    @Autowired OrderLineRepository orderLineRepository;
    @Autowired DishRepository dishRepository;
    @Autowired RestaurantRepository restaurantRepository;

    Restaurant restaurant;
    Dish ensalada;
    Dish lentejas;
    Dish tarta;
    Order pedido;

    @BeforeEach
    void setUp() {
        restaurant = restaurantRepository.save(
            Restaurant.builder().name("La Taberna").build()
        );
        ensalada = dishRepository.save(Dish.builder().name("Ensalada").price(5.0).restaurant(restaurant).build());
        lentejas = dishRepository.save(Dish.builder().name("Lentejas").price(8.0).restaurant(restaurant).build());
        tarta = dishRepository.save(Dish.builder().name("Tarta").price(7.50).restaurant(restaurant).build());

        pedido = orderRepository.save(
            Order.builder().status(OrderStatus.PENDING).restaurant(restaurant).build()
        );
    }

    @Test
    @DisplayName("recalculateTotal suma 2×5 + 1×8 + 3×7.50 = 40.50 y lo guarda")
    void recalculateTotal() {
        // Given
        orderLineRepository.saveAll(List.of(
            OrderLine.builder().quantity(2).dish(ensalada).order(pedido).build(),
            OrderLine.builder().quantity(1).dish(lentejas).order(pedido).build(),
            OrderLine.builder().quantity(3).dish(tarta).order(pedido).build()
        ));

        // When
        Order actualizado = orderService.recalculateTotal(pedido.getId());

        // Then
        assertEquals(40.50, actualizado.getTotalPrice());
    }

    @Test
    @DisplayName("recalculateTotal con pedido sin líneas deja total en 0")
    void recalculateTotal_sinLineas() {
        // When
        Order actualizado = orderService.recalculateTotal(pedido.getId());

        // Then
        assertEquals(0.0, actualizado.getTotalPrice());
    }

    @Test
    @DisplayName("addDish añade una línea y actualiza el total del pedido")
    void addDish() {
        // When - añadir 2 ensaladas
        Order despuesDeEnsalada = orderService.addDish(pedido.getId(), ensalada.getId(), 2);

        // Then - total = 2 × 5 = 10
        assertEquals(10.0, despuesDeEnsalada.getTotalPrice());

        // When - añadir 1 lentejas
        Order despuesDeLentejas = orderService.addDish(pedido.getId(), lentejas.getId(), 1);

        // Then - total = 10 + 8 = 18
        assertEquals(18.0, despuesDeLentejas.getTotalPrice());

        // When - añadir 3 tartas
        Order despuesDeTarta = orderService.addDish(pedido.getId(), tarta.getId(), 3);

        // Then - total = 18 + 22.50 = 40.50
        assertEquals(40.50, despuesDeTarta.getTotalPrice());
    }

    @Test
    @DisplayName("recalculateTotal con pedido inexistente lanza excepción")
    void recalculateTotal_pedidoNoExiste() {
        assertThrows(RuntimeException.class, () -> orderService.recalculateTotal(9999L));
    }
}
```

### Diferencias entre tests de repo y tests de service

| | @DataJpaTest (repo) | @SpringBootTest (service) |
|---|---|---|
| Qué arranca | Solo capa JPA | Toda la app |
| Rollback automático | Sí | Hay que poner `@Transactional` |
| Qué testea | Una operación SQL | Varios repos coordinados + lógica |
| Velocidad | Rápido | Más lento |
