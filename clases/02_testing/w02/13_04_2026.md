## Aislamiento entre tests
* Regla: los tests REST que mutan la BD deben rollbackear para no contaminar Selenium / Playwright
* En `@DataJpaTest` el rollback es automático
* En `@SpringBootTest` con MockMvc tendremos que poner `@Transactional` a mano
* Cada test es independiente, nunca depender de que otro se ejecute antes

## Refactor con @BeforeEach
Extraer la creación de objetos de prueba a un método `setUp()`:

```java
@BeforeEach
void setUp() {
    restaurant1 = Restaurant.builder().name("R1").averagePrice(20.0).active(true).build();
    restaurant2 = Restaurant.builder().name("R2").averagePrice(15.0).active(true).build();
}
```
