## DataInitializer: cargar datos al arrancar

Antes de los controladores, necesitamos datos para mostrar en el navegador. Movemos la creación de datos a una clase que se ejecuta al arrancar:

Nuevo archivo: `src/main/java/com/restaurantes/config/DataInitializer.java`

```java
package com.restaurantes.config;

import com.restaurantes.model.*;
import com.restaurantes.repository.*;
import org.springframework.boot.CommandLineRunner;
import org.springframework.context.annotation.Profile;
import org.springframework.stereotype.Component;

import java.time.LocalDateTime;
import java.util.List;

@Component
@Profile("!test")
public class DataInitializer implements CommandLineRunner {

    private final RestaurantRepository restaurantRepository;
    private final DishRepository dishRepository;
    private final OrderRepository orderRepository;
    private final OrderLineRepository orderLineRepository;

    public DataInitializer(RestaurantRepository restaurantRepository,
                           DishRepository dishRepository,
                           OrderRepository orderRepository,
                           OrderLineRepository orderLineRepository) {
        this.restaurantRepository = restaurantRepository;
        this.dishRepository = dishRepository;
        this.orderRepository = orderRepository;
        this.orderLineRepository = orderLineRepository;
    }

    @Override
    public void run(String... args) {
        Restaurant taberna = restaurantRepository.save(
            Restaurant.builder().name("La Taberna").averagePrice(15.0).active(true).foodType(FoodType.SPANISH).build()
        );
        Restaurant dominos = restaurantRepository.save(
            Restaurant.builder().name("Dominos Pizza").averagePrice(12.0).active(true).build()
        );

        Dish ensalada = dishRepository.save(Dish.builder().name("Ensalada").price(5.0).restaurant(taberna).build());
        Dish lentejas = dishRepository.save(Dish.builder().name("Lentejas").price(8.0).restaurant(taberna).build());
        Dish tarta = dishRepository.save(Dish.builder().name("Tarta de queso").price(7.50).restaurant(taberna).build());

        Order pedido = orderRepository.save(
            Order.builder().date(LocalDateTime.now()).tableNumber(5).numPeople(3)
                .status(OrderStatus.PENDING).restaurant(taberna).build()
        );
        orderLineRepository.saveAll(List.of(
            OrderLine.builder().quantity(2).dish(ensalada).order(pedido).build(),
            OrderLine.builder().quantity(1).dish(lentejas).order(pedido).build(),
            OrderLine.builder().quantity(3).dish(tarta).order(pedido).build()
        ));

        Double total = orderLineRepository.calculateTotalPrice(pedido.getId());
        pedido.setTotalPrice(total);
        orderRepository.save(pedido);
    }
}
```

* `@Component` — Spring ejecuta esta clase al arrancar
* `@Profile("!test")` — **NO** se ejecuta durante los tests. Así los tests siguen arrancando con BD vacía
* `CommandLineRunner` — el método `run()` se ejecuta al iniciar la app

## Primer controlador: RestaurantController

Nuevo archivo: `src/main/java/com/restaurantes/controller/RestaurantController.java`

```java
package com.restaurantes.controller;

import com.restaurantes.repository.RestaurantRepository;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class RestaurantController {

    private final RestaurantRepository restaurantRepository;

    public RestaurantController(RestaurantRepository restaurantRepository) {
        this.restaurantRepository = restaurantRepository;
    }

    @GetMapping("/restaurants")
    public String findAll(Model model) {
        model.addAttribute("restaurants", restaurantRepository.findAll());
        return "restaurant-list";
    }
}
```

* `@Controller` — recibe peticiones HTTP del navegador
* `@GetMapping("/restaurants")` — responde cuando alguien navega a `localhost:8080/restaurants`
* `Model model` — caja donde metemos datos para enviar al HTML
* `return "restaurant-list"` — Spring busca `templates/restaurant-list.html`

## Testear el controlador con MockMvc

Ahora la parte nueva: **testear que el controlador funciona correctamente sin abrir el navegador**.

Nuevo archivo: `src/test/java/com/restaurantes/controller/RestaurantControllerTest.java`

```java
package com.restaurantes.controller;

import com.restaurantes.model.Restaurant;
import com.restaurantes.repository.RestaurantRepository;
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.AutoConfigureMockMvc;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.test.web.servlet.MockMvc;
import org.springframework.transaction.annotation.Transactional;

import static org.hamcrest.Matchers.*;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.*;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.*;

@SpringBootTest
@AutoConfigureMockMvc
@Transactional
class RestaurantControllerTest {

    @Autowired
    MockMvc mockMvc;

    @Autowired
    RestaurantRepository restaurantRepository;

    @Test
    @DisplayName("GET /restaurants devuelve HTTP 200 y usa la vista restaurant-list")
    void findAll() throws Exception {
        // Given
        restaurantRepository.save(Restaurant.builder().name("Test 1").averagePrice(10.0).build());
        restaurantRepository.save(Restaurant.builder().name("Test 2").averagePrice(20.0).build());

        // When & Then
        mockMvc.perform(get("/restaurants"))
                .andExpect(status().isOk())
                .andExpect(view().name("restaurant-list"))
                .andExpect(model().attributeExists("restaurants"))
                .andExpect(model().attribute("restaurants", hasSize(2)));
    }

    @Test
    @DisplayName("GET /restaurants con BD vacía devuelve lista vacía")
    void findAll_vacio() throws Exception {
        mockMvc.perform(get("/restaurants"))
                .andExpect(status().isOk())
                .andExpect(view().name("restaurant-list"))
                .andExpect(model().attribute("restaurants", hasSize(0)));
    }
}
```

### Conceptos clave de MockMvc

| Concepto | Qué hace |
|---|---|
| `@SpringBootTest` | Arranca toda la aplicación Spring (no solo JPA como `@DataJpaTest`) |
| `@AutoConfigureMockMvc` | Configura `MockMvc` para poder simular peticiones HTTP |
| `@Transactional` | Rollback automático después de cada test (como en `@DataJpaTest`) |
| `MockMvc mockMvc` | Simula un navegador: hace peticiones HTTP sin abrir Chrome |
| `mockMvc.perform(get("/restaurants"))` | Simula que el usuario navega a `/restaurants` |
| `status().isOk()` | Comprueba que el servidor responde con HTTP 200 (éxito) |
| `view().name("restaurant-list")` | Comprueba que el controlador devuelve la vista correcta |
| `model().attributeExists("restaurants")` | Comprueba que el modelo tiene el atributo `restaurants` |
| `model().attribute("restaurants", hasSize(2))` | Comprueba que la lista tiene 2 elementos |

### Comparación con @DataJpaTest

| | @DataJpaTest | @SpringBootTest + MockMvc |
|---|---|---|
| Qué arranca | Solo la capa JPA (repos) | Toda la app (controller + repo + config) |
| Qué testea | Operaciones de base de datos | Peticiones HTTP al controlador |
| Rollback automático | Sí | Hay que poner `@Transactional` |
| `@Profile("!test")` | DataInitializer NO se carga (no es `@DataJpaTest` aware) | DataInitializer NO se carga (por el `@Profile("!test")`) |
