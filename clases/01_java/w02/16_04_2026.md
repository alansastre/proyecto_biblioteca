## 2. Entidad Order (pedido)

```java
package com.restaurantes.model;

import jakarta.persistence.*;
import java.time.LocalDateTime;

@Entity
@Table(name = "orders")
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private LocalDateTime date;

    private Double totalPrice;

    private Double tip;

    private Integer tableNumber;

    private Integer numPeople;

    @Enumerated(EnumType.STRING)
    private OrderStatus status;

    @ManyToOne
    private Restaurant restaurant;

    // Constructor vacío
    public Order() {}

    // Constructor con todos los parámetros excepto id
    public Order(LocalDateTime date, Double totalPrice, Double tip,
                 Integer tableNumber, Integer numPeople,
                 OrderStatus status, Restaurant restaurant) {
        this.date = date;
        this.totalPrice = totalPrice;
        this.tip = tip;
        this.tableNumber = tableNumber;
        this.numPeople = numPeople;
        this.status = status;
        this.restaurant = restaurant;
    }

    // Getter y Setter

    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    }

    public LocalDateTime getDate() {
        return date;
    }

    public void setDate(LocalDateTime date) {
        this.date = date;
    }

    public Double getTotalPrice() {
        return totalPrice;
    }

    public void setTotalPrice(Double totalPrice) {
        this.totalPrice = totalPrice;
    }

    public Double getTip() {
        return tip;
    }

    public void setTip(Double tip) {
        this.tip = tip;
    }

    public Integer getTableNumber() {
        return tableNumber;
    }

    public void setTableNumber(Integer tableNumber) {
        this.tableNumber = tableNumber;
    }

    public Integer getNumPeople() {
        return numPeople;
    }

    public void setNumPeople(Integer numPeople) {
        this.numPeople = numPeople;
    }

    public OrderStatus getStatus() {
        return status;
    }

    public void setStatus(OrderStatus status) {
        this.status = status;
    }

    public Restaurant getRestaurant() {
        return restaurant;
    }

    public void setRestaurant(Restaurant restaurant) {
        this.restaurant = restaurant;
    }

    @Override
    public String toString() {
        return "Order{" +
                "id=" + id +
                ", date=" + date +
                ", totalPrice=" + totalPrice +
                ", tip=" + tip +
                ", tableNumber=" + tableNumber +
                ", numPeople=" + numPeople +
                ", status=" + status +
                ", restaurant=" + (restaurant != null ? restaurant.getName() : null) +
                '}';
    }
}
```

Puntos clave:
* `@Table(name = "orders")` — la palabra `order` es reservada en SQL (`ORDER BY`), si no la renombramos da error
* `LocalDateTime` — incluye fecha y hora (`2026-04-16T14:30:00`), a diferencia de `LocalDate` que solo tiene fecha
* `@ManyToOne` con Restaurant — un pedido pertenece a un restaurante, muchos pedidos pueden ser del mismo restaurante
* En el `toString` usamos `restaurant != null ? restaurant.getName() : null` para evitar `NullPointerException`, igual que hicimos con Employee

## 3. Entidad OrderLine (línea de pedido)

Un pedido tiene varias líneas. Cada línea indica qué plato se pide y cuántas unidades:

```
Pedido mesa 5:
  - 2x Ensalada (5.00 €)    → línea 1
  - 1x Lentejas (8.00 €)    → línea 2
  - 3x Tarta de queso (7.50 €) → línea 3
```

```java
package com.restaurantes.model;

import jakarta.persistence.*;

@Entity
public class OrderLine {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Integer quantity;

    @ManyToOne
    private Dish dish;

    @ManyToOne
    private Order order;

    // Constructor vacío
    public OrderLine() {}

    // Constructor con parámetros
    public OrderLine(Integer quantity, Dish dish, Order order) {
        this.quantity = quantity;
        this.dish = dish;
        this.order = order;
    }

    // Getter y Setter

    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    }

    public Integer getQuantity() {
        return quantity;
    }

    public void setQuantity(Integer quantity) {
        this.quantity = quantity;
    }

    public Dish getDish() {
        return dish;
    }

    public void setDish(Dish dish) {
        this.dish = dish;
    }

    public Order getOrder() {
        return order;
    }

    public void setOrder(Order order) {
        this.order = order;
    }

    @Override
    public String toString() {
        return "OrderLine{" +
                "id=" + id +
                ", quantity=" + quantity +
                ", dish=" + (dish != null ? dish.getName() : null) +
                ", orderId=" + (order != null ? order.getId() : null) +
                '}';
    }
}
```

Puntos clave:
* Tiene **dos** `@ManyToOne`: una hacia `Dish` (qué plato) y otra hacia `Order` (a qué pedido pertenece)
* `quantity` indica cuántas unidades de ese plato se piden
* Es la primera vez que una entidad tiene dos asociaciones `@ManyToOne` — es totalmente válido

## 4. Repositorios

```java
package com.restaurantes.repository;

import com.restaurantes.model.Order;
import org.springframework.data.jpa.repository.JpaRepository;

public interface OrderRepository extends JpaRepository<Order, Long> {
}
```

```java
package com.restaurantes.repository;

import com.restaurantes.model.OrderLine;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;

import java.util.List;

public interface OrderLineRepository extends JpaRepository<OrderLine, Long> {

    // Traer todas las líneas de un pedido por su id
    List<OrderLine> findByOrderId(Long orderId);

    // Calcular el precio total de un pedido directamente en base de datos
    @Query("SELECT SUM(ol.quantity * ol.dish.price) FROM OrderLine ol WHERE ol.order.id = ?1")
    Double calculateTotalPrice(Long orderId);
}
```

* `findByOrderId` — consulta derivada que ya conocemos: filtra las líneas que pertenecen a un pedido concreto
* `calculateTotalPrice` — aquí no podemos usar un nombre derivado porque Spring Data no sabe generar automáticamente una suma con multiplicación. Por eso usamos `@Query` para escribir la consulta nosotros

### @Query: consultas personalizadas

Hasta ahora hemos visto dos tipos de consultas en repositorios:

| Tipo | Ejemplo | Cuándo usarlo |
|---|---|---|
| Heredado | `findAll()`, `findById()`, `save()` | Operaciones CRUD básicas |
| Derivado | `findByRestaurantName()`, `findByAgeGreaterThanEqual()` | Filtros simples basados en atributos |
| **@Query** | `calculateTotalPrice()` | **Consultas complejas: sumas, agrupaciones, joins con cálculos** |

La anotación `@Query` usa **JPQL** (Java Persistence Query Language), que es parecido a SQL pero usa los nombres de las clases Java en vez de las tablas:

```
SQL:   SELECT SUM(ol.quantity * d.price) FROM order_line ol JOIN dish d ON ol.dish_id = d.id WHERE ol.order_id = 1
JPQL:  SELECT SUM(ol.quantity * ol.dish.price) FROM OrderLine ol WHERE ol.order.id = ?1
```

Diferencias con SQL:
* Usamos `OrderLine` (nombre de la clase) en vez de `order_line` (nombre de la tabla)
* Usamos `ol.dish.price` (navegando por la asociación) en vez de hacer un `JOIN` manual
* `?1` es el primer parámetro del método (`Long orderId`)

## 7. Calcular el precio total del pedido

El precio total es la suma de `cantidad × precio` de cada línea:

```
2 × 5.00 (Ensalada)     = 10.00
1 × 8.00 (Lentejas)     =  8.00
3 × 7.50 (Tarta queso)  = 22.50
                    TOTAL = 40.50
```

En código, después de crear las líneas:

```java
// Calcular precio total del pedido
List<OrderLine> lineasPedido = orderLineRepository.findByOrderId(pedido1.getId());

double total = 0;
for (OrderLine linea : lineasPedido) {
    double precioLinea = linea.getQuantity() * linea.getDish().getPrice();
    total = total + precioLinea;
}

// Guardar el total en el pedido
pedido1.setTotalPrice(total);
orderRepository.save(pedido1);

System.out.println("Precio total del pedido: " + total + " €");
```

Paso a paso:
* Traemos las líneas del pedido con `findByOrderId`
* Recorremos cada línea con un `for`
* Para cada línea: `cantidad × precio del plato` = precio de esa línea
* Sumamos todas las líneas en la variable `total`
* Guardamos el total en el pedido con `setTotalPrice` y `save`

### Opción 2: calcular con @Query desde el repositorio

En vez de traer todas las líneas a Java y sumar con un bucle, podemos hacer que la base de datos calcule la suma directamente:

```java
// Calcular precio total usando @Query (la base de datos hace el cálculo)
Double totalDesdeDB = orderLineRepository.calculateTotalPrice(pedido1.getId());
pedido1.setTotalPrice(totalDesdeDB);
orderRepository.save(pedido1);

System.out.println("Precio total (calculado por la BD): " + totalDesdeDB + " €");
```

Comparación de las dos opciones:

| | Opción 1: bucle for en Java | Opción 2: @Query en repositorio |
|---|---|---|
| Código | Más largo, bucle con variable acumuladora | Una sola línea |
| Quién calcula | Java (la aplicación) | La base de datos |
| Datos que viajan | Trae TODAS las líneas a Java | Solo trae el número resultado |

Las dos opciones dan el mismo resultado (`40.50`). La opción con `@Query` es más eficiente porque la base de datos solo devuelve un número en vez de todas las filas.
